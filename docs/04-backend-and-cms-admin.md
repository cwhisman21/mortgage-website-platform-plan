# Backend And CMS Admin

## Admin Principle

The admin should manage structured mortgage entities and relationships, not just rich-text pages.

The admin is a mortgage operating CMS:

- Content.
- Market data.
- SEO.
- Compliance.
- Local placement.
- Lead/opportunity routing.

## Admin Areas

Recommended admin sidebar:

- Dashboard.
- Locations.
- Loan Officers.
- Branches.
- Products.
- Blog / News.
- Market Data.
- Page Templates.
- CTA Rules.
- SEO.
- Compliance.
- Leads / Opportunities.
- AI Writer Briefs.
- Settings.

## Locations Admin

Each city/state should support:

- Basic info: name, slug, parent state, county/metro, status.
- Market facts: prices, taxes, insurance notes, affordability notes.
- Relationships: branches, loan officers, products, articles, calculators.
- Page modules: charts, tables, news feed, FAQ, CTA behavior.
- AI writer brief: content angles, required facts, preferred internal links.

## Loan Officer Admin

Each loan officer should support:

- Profile: name, title, photo, NMLS, phone, email, bio.
- Licensing: licensed states, office, service areas.
- Placement: priority markets, specialties, languages, product fit.
- Content: articles, featured quotes, testimonials, FAQs.
- CTA links: apply link, calendar link, contact route.

## Product Admin

Each product should support:

- Product summary.
- Borrower fit.
- Requirements.
- Benefits and tradeoffs.
- FAQs.
- Related calculators.
- Related products.
- Specialist loan officers.
- Localized variants.

## Editorial Admin

Articles should include:

- Type: local news, market update, product guide, buyer education, refinance guide, tax/insurance explainer.
- Relationships: locations, products, loan officers, branches, calculators.
- Workflow: draft, AI drafted, editor reviewed, compliance reviewed, published, archived.
- Source notes and last-reviewed date.

## CTA Admin

CTA rules should be configurable by:

- Page type.
- Location.
- Product.
- User intent.
- Scroll section.
- Device.

Example:

- City page hero: Get prequalified.
- Market data section: Estimate payment.
- Loan officer section: Choose a local expert.
- Product section: Compare loan options.

## Compliance Admin

Required controls:

- NMLS records.
- Equal Housing language.
- State licensing disclosures.
- Rate/APR disclaimers.
- Product disclaimers.
- Article review status.
- Version history.
- Publish approvals.
- Archived page versions.
